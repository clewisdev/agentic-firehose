# Harness engineering

The infrastructure around an AI model — personalisation, delegation, memory, context, and action — as the primary determinant of production system quality. Era 3 of AI engineering (Hashimoto, 2026).

## Sources

- [Harness Engineering Eras](../../sources/2026-06-05-harness-engineering-eras.md) — three-era framing: prompt → context → harness; "The benchmarks are virtually identical across LLMs. What separates outputs now is the harness you build." Five harness components: personalisation, delegation, memory, context, action.
- [Pydantic AI Harness — Production](../../sources/2026-05-31-pydantic-ai-harness-production.md) — practitioner confirmation: "The model is no longer the product... The harness is." Capability matrix (execution, context, memory, orchestration, reliability, safety).
