# Tool use

How agents call tools, how tool calls and results flow through the loop, and patterns for designing tool surfaces (token efficiency, error shapes, batching, retrieval-shaped APIs).

## Sources

- [Böckeler — Maintainability sensors for coding agents](../../sources/2026-05-30-fowler-maintainability-sensors.md) — static analysis tools (ESLint, dependency-cruiser, Stryker) used as in-loop agent feedback mechanisms. Custom CLI to query Stryker JSON rather than raw file avoids context overflow — "AI helping AI." Key design: tools that produce agent-readable self-correction guidance, not just machine output.

- [claude-mem](../../sources/2026-05-20-claude-mem.md) — observes `PostToolUse` to build an external log of every tool call. Exposes retrieval as MCP tools shaped in 3 layers (compact index → context → full body) for token efficiency.
- [Google agent skills](../../sources/2026-05-28-google-skills-repo.md) — vendor-published compact skills for GCP APIs. Demonstrates agent-first documentation format as the alternative to feeding full API docs into context. Skill Registry API suggests runtime-discoverable tools.
- [I Don't Want an AI God](../../sources/2026-05-30-ai-god-agent-orchestration.md) — unified capture via Telegram/Siri/Gmail/feeds; worker loop picks one task per invocation and exits without cross-project context — minimal tool exposure per task as a design principle.
- [Codex-maxxing](../../sources/2026-05-31-codex-maxxing.md) — scoped tool access by context type ($browser, @chrome, @computer) prevents mode confusion; each context surface has its own tool namespace.
- [Pydantic AI Harness — Production](../../sources/2026-05-31-pydantic-ai-harness-production.md) — MCP and CodeMode (sandboxed Python) as tool-access harness modules; excessive agency risk when tool surface is not deliberately scoped.
- [Packaging Copilot Agents & Skills with APM](../../sources/2026-06-13-packaging-github-copilot-apm.md) — scenario packages (architect / terraform / diagramming) scope tool surface per workflow, including the MCPs each needs (Draw.io, Excalidraw); "too much context is not neutral" — narrow tool exposure as a deliberate package boundary.
- [Harness engineering: Codex in an agent-first world (OpenAI)](../../sources/2026-06-13-harness-engineering-codex.md) — make app state queryable as tools: Chrome DevTools Protocol + DOM snapshots for UI legibility; observability stack (logs/metrics/traces) Codex queries directly via LogQL/PromQL. Agents reproduce and validate bugs without human copy-paste; tools designed for agent self-service, not human dashboards.
- [What it feels like to work with Mythos (Mollick)](../../sources/2026-06-11-what-it-feels-like-to-work-with-mythos.md) — agent-driven tool use at scale: Fable retrieved 2,200+ real flights, TGV/Shinkansen rail schedules, and per-country road speeds from academic papers by orchestrating sub-agents — no human wiring the queries. Heavy token spend as the cost of autonomous data gathering.
- [Claude Fable 5 & Mythos 5 (Anthropic)](../../sources/2026-06-10-claude-fable-5-mythos-5.md) — vision as a first-class tool input: code reconstruction from screenshots, Pokémon/Slay-the-Spire play from raw game state, scientific-figure extraction. Vision is now primary input modality for complex tasks, not secondary annotation.
- [Claude Code security plugin (Anthropic)](../../sources/2026-06-09-claude-security-plugin.md) — tool surface as enforcement point: review layers hook the edit / end-of-turn / commit-push (Bash) lifecycle; the per-edit pattern-match layer adds zero usage cost; custom rules via `.claude/security-patterns.yaml`. Example of instrumenting the tool loop rather than the model.
- [Agent Reach: Give your AI agent eyes to see the entire internet](../../sources/2025-06-10-agent-reach.md) — scaffolding, not a framework: gives agents web reach across 15+ platforms by installing and orchestrating existing CLI tools (twitter-cli, yt-dlp, gh, Jina Reader) + MCP (free Exa search). Each channel is an independent upstream tool — swap any without lock-in.
- [Introducing dynamic workflows in Claude Code](../../sources/2025-06-09-dynamic-workflows-claude-code.md) — `ultracode` mode (effort xhigh) lets Claude decide when to spin up a workflow and write the orchestration scripts itself. First trigger shows what will run and requires confirmation; admins can disable via managed settings.
- [Agent Skills Overview - Standardized Format for AI Agent Capabilities](../../sources/2026-06-14-agent-skills-overview.md) — skills package procedural knowledge + executable code + references as portable, version-controlled folders agents load on demand; one definition reused across agent products and clients.
- [Claude Code's creator on moving beyond prompting to loops and agentic workflows](../../sources/2026-06-12-claude-code-loops-abstraction.md) — Claude Code built (late 2024) to close 'product overhang' — models could do more than tools exposed, and coding was the obvious gap. Dev tools work best as a side focus on real business problems, not a product obsession.
- [Obsidian-Second-Brain: Agentic Memory via Vault Mutation](../../sources/2026-06-09-obsidian-second-brain-agent-memory.md) — Agent as active memory custodian: /obsidian-reconcile and /obsidian-challenge as tools that mutate the vault rather than appending to it
- [LangChain's Deep Agents: Open-source coding agent framework](../../sources/2026-06-08-deepagents-langchain-coding.md) — Model-agnostic design: orchestration patterns (planning, sub-agent handoffs, context management) as the framework value, not LLM coupling — swap models without changing the tool layer
- [Claude Code Docs – Overview](../../sources/2026-06-15-claude-code-docs-overview.md) — Anthropic official docs: CLAUDE.md + auto-memory for context persistence; MCP as first-class extension point; multi-platform (terminal/IDE/desktop/web); multi-agent lead-coordinator pattern with Agent SDK for custom orchestration
- [last30days: Real-time research aggregator for pre-call context](../../sources/2025-06-12-last30days-client-prep.md) — last30days as agent tool: zero-config for Reddit/HN/GitHub/Polymarket; 30s setup for gated APIs (X, YouTube); agent accesses sources no single model has native access to
- [AI Engineering Coach – VS Code Extension for AI Session Analysis](../../sources/2025-06-14-ai-engineering-coach.md) — Microsoft AI Engineering Coach (VS Code): 45 editable rules across prompt quality, session hygiene, code review, tool mastery, context management; local-only analysis; Skill Finder bridges individual practice to community knowledge
- [Cowork: How to Set Up Claude for Non-Coders](../../sources/2026-06-09-cowork-claude-setup-guide.md) — Hassid: Cowork folder architecture for non-coders — about-me.md (~2k tokens), anti-ai-writing-style.md; interview-driven extraction of patterns, not raw transcripts; 'every sentence should carry signal'
