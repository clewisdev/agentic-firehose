# Engineering judgment

The human judgment layer that AI amplifies rather than replaces — system design decisions, trade-off understanding, knowing when AI is wrong, and risk acceptance at merge time.

## Sources

- [AI sprinkled on top of a broken software lifecycle](../../sources/2026-05-31-ai-sprinkled-broken-software.md) — "Typing code is becoming less valuable. Engineering judgment is becoming more valuable." Code review, system design, maintaining standards, knowing when AI is wrong.
- [Building OpenCode with Dax Raad](../../sources/2026-06-02-opencode-podcast-dax-raad.md) — cognitive load doesn't decrease proportionally: 95% thinking / 5% doing pre-AI → 96% / 4% post-AI; "a 20% improvement on the 5%, but it feels as hard as ever." Vendor narratives of dramatic productivity gains are contested by practitioners.
- [Is this sustainable? (Hurst)](../../sources/2026-06-09-ai-sustainable-hurst.md) — why AI landed on senior roles first: "seniority is where the system-level understanding lives," so seniors recognise where AI applies across the SDLC *and* build it. Judgment is the multiplier that lets one person do a team's work — and the scarce input that doesn't scale with tooling.
- [Mark Richards and Neal Ford Reflect on Software Architecture: The Hard Parts](../../sources/2026-06-10-mark-richards-neal-ford-software-architecture-hard-parts.md) — 'they can find recipes, but they can't reason' — architectural tradeoff reasoning is fundamentally different from recipe-following, and closing that gap is the hard part. A pointed frame for what AI codegen (great at recipes) doesn't yet replace.
  - [Will There Be Source Code in the Future?](../../sources/2026-06-09-will-there-be-source-code-future.md) — Naur's 'Programming as Theory Building' (1985) revived: the real cost is holding the 'why' in human minds, not writing instructions. 'The act of writing code IS the thinking' — removing the synthesis work may remove the mechanism that builds understanding.

## Synthesis

Writing code was load-bearing for theory-building.
