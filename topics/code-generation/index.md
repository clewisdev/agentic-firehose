# Code generation

Using AI to generate implementation from specifications — the abstraction ladder (code-first→spec-first), when generation is deterministic vs. requires agent creativity, and the boundary where generation breaks down.

## Synthesis

- [The Verification Bottleneck: When Generation Outran Review](../../synthesis/verification-bottleneck.md) (2026-06-15, draft) — synthesised across 11 sources: generation got cheap (Codex ~1M lines zero hand-written; Bun 750K lines in 11 days), but "throughput measures what was shipped, not what survived." The constraint moved downstream to verification and review.

## Sources

- [The Spec Is the Code Now](../../sources/2026-06-04-spec-is-code-now.md) — "We stopped writing assembly by hand. We may stop writing most implementation by hand too." Spec-first as the infrastructure play that makes code generation viable at scale; guesswork disappears with contract clarity on inputs, types, error codes.
- [Harness engineering: Codex in an agent-first world (OpenAI)](../../sources/2026-06-13-harness-engineering-codex.md) — concrete proof point: ~1M lines, zero hand-written code, 3 engineers at 3.5 merged PRs/day each (throughput rose as the team grew to 7). Intent specification scales faster than code review; the bottleneck moves from generation to human QA capacity — solved by making app behaviour agent-legible.
- [The AI Engineering Report 2026 — Ten Takeaways](../../sources/2026-06-12-ai-engineering-report-2026.md) — the counterweight to Codex's success story: AI is now the *primary* code author (acceptance 20%→60%), but bugs/developer rose 9%→54% and the defect relationship steepens with adoption maturity. "Throughput measures what was shipped, not what survived."
- [What it feels like to work with Mythos (Mollick)](../../sources/2026-06-11-what-it-feels-like-to-work-with-mythos.md) — generation from a 19-page self-authored design doc, executed autonomously over 9.5h ("Concord"). Output scope "exceeded anything I have seen before," but the expert can only validate the final artifact — generation outruns reviewability.
- [Claude Code security plugin (Anthropic)](../../sources/2026-06-09-claude-security-plugin.md) — agent-as-reviewer for generated code: a fresh-context Claude instance (no investment in the original approach) reviews the diff, avoiding the author model's confirmation bias and catching logic-level vulns string matching misses.
- [Agent Skills: Structured Workflows for Production-Grade AI Code Generation](../../sources/2025-06-11-agent-skills-structured-workflows.md) — the prototype→production gap: agents hallucinate technical debt and skip tests/specs because they optimize for appearing done over being done. Code-as-liability: prioritize what must not break over what ships fast.
- [Introducing dynamic workflows in Claude Code](../../sources/2025-06-09-dynamic-workflows-claude-code.md) — Bun ported Zig→Rust via dynamic workflows: 750K lines of Rust generated, 99.8% of the test suite passing, 11 days — two reviewers per file plus a fix loop driving build/test to clean. 'Work you'd plan in quarters now finishes in days,' at substantially higher token cost.
- [Context & Harness Engineering - The Transition to Agentic Software Delivery](../../sources/2026-06-14-context-harness-engineering.md) — input quality (context engineering) and output quality (harness engineering) are orthogonal but mutually reinforcing; AI adoption without governance *accelerates* technical-debt accumulation, not slows it.
- [Context as Code: Build-time Governance in the Era of Infinite Syntax](../../sources/2026-06-13-context-as-code.md) — syntax is cheap, architectural control is scarce. 'Frankenstein factories' accrue compounding comprehension debt (Osmani) when agents generate faster than teams verify intent — the rare resource is assurance, not output.
- [Fabro: Workflow Graphs for AI Agent Orchestration](../../sources/2026-06-12-fabro-workflow-orchestration.md) — produces 'reviewable changes, not massive diffs' — output you can trust because each stage commits and validates, instead of an opaque agent emitting a 50-file diff.
- [Augmented Coding Weekly #48: Claude Fable 5, AI safety, and the rockstar developer problem](../../sources/2026-06-12-augmented-coding-weekly-48.md) — concrete win + caution: a 50M-line Ruby codebase migrated in a day (vs ~two months by a team), but Eberhardt flags a velocity gap — spec-light/flexible apps gain massively while requirements-heavy professional engineering sees diminishing returns over Opus.
- [Will There Be Source Code in the Future?](../../sources/2026-06-09-will-there-be-source-code-future.md) — code's two roles — machine instructions and conceptual model — split as AI handles synthesis: the instruction role cheapens, the modeling role becomes central. 'The mechanical instructions got cheap, the theory of why a system is the way it is didn't.'
- [Supervisory engineering: Orchestrating software's 'middle loop'](../../sources/2026-06-09-supervisory-engineering-middle-loop.md) — 'The bottleneck is no longer how fast we can type, it's how fast we can verify' — names the middle loop as the structural response to AI collapsing the inner loop
