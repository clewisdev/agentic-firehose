# Enterprise deployment

Deploying AI agents in regulated, high-stakes enterprise environments — Model Risk Management (MRM), architectural discipline, governance, and the mismatch between agent hype and production-grade deployment requirements.

## Sources

- [MSFT Build AI SDLC](../../sources/2026-06-05-msft-build-ai-sdlc.md) — financial services as forcing function (10k–30k releases/year bar exposes every immaturity); "agent volume is not a competitive advantage" without integration into robust SDLC; low-code platforms create production debt in regulated industries.
- [The AI Engineering Report 2026 — Ten Takeaways](../../sources/2026-06-12-ai-engineering-report-2026.md) — telemetry from 22k developers / 4,000+ teams (not sentiment): "Acceleration Whiplash." Real gains (epics/dev +66%, throughput +33.7%) shadowed by incidents-per-PR +242.7% and code churn +861%. Org-level evidence that human-paced delivery systems can't safely absorb agent output.
- [Is this sustainable? (Hurst)](../../sources/2026-06-09-ai-sustainable-hurst.md) — "the cost of building has collapsed, but the cost of aligning organisationally has not." When multiple teams can independently ship the same solution faster, coordination/consolidation becomes the bottleneck; skills redistribution toward effective tool-adopters happens without governance.
