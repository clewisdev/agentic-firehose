# Enterprise deployment

Deploying AI agents in regulated, high-stakes enterprise environments — Model Risk Management (MRM), architectural discipline, governance, and the mismatch between agent hype and production-grade deployment requirements.

## Sources

- [MSFT Build AI SDLC](../../sources/2026-06-05-msft-build-ai-sdlc.md) — financial services as forcing function (10k–30k releases/year bar exposes every immaturity); "agent volume is not a competitive advantage" without integration into robust SDLC; low-code platforms create production debt in regulated industries.
- [The AI Engineering Report 2026 — Ten Takeaways](../../sources/2026-06-12-ai-engineering-report-2026.md) — telemetry from 22k developers / 4,000+ teams (not sentiment): "Acceleration Whiplash." Real gains (epics/dev +66%, throughput +33.7%) shadowed by incidents-per-PR +242.7% and code churn +861%. Org-level evidence that human-paced delivery systems can't safely absorb agent output.
- [Is this sustainable? (Hurst)](../../sources/2026-06-09-ai-sustainable-hurst.md) — "the cost of building has collapsed, but the cost of aligning organisationally has not." When multiple teams can independently ship the same solution faster, coordination/consolidation becomes the bottleneck; skills redistribution toward effective tool-adopters happens without governance.
- [Claude Fable 5 Launch: Mythos-class Model with Safety Fallback Architecture](../../sources/2025-06-10-claude-fable-5-announcement.md) — Anthropic reports 8x internal code velocity, pushing the bottleneck from generation to verification & review. Bio/cyber use excluded with a vague access path — a deployment liability. 'Knowing when not to answer' framed as a design principle, not a limitation.
- [What Happens to Platform Teams?](../../sources/2026-06-15-platform-teams-ai-era.md) — platform teams in the AI era: the 'build-once-centrally' justification erodes as teams get 'AI-bulldozers' to build bespoke implementations in a day — but the cognitive-load justification strengthens as surface area multiplies. Operational responsibility becomes the scarce resource.
- [Context & Harness Engineering - The Transition to Agentic Software Delivery](../../sources/2026-06-14-context-harness-engineering.md) — board-level primer: ungoverned AI codegen violates emerging regulatory frameworks (EU AI Act, ISO 42001, FCA, SEC) and accumulates security/technical debt at machine speed. Six questions for CTOs; diffuse governance ownership creates accountability vacuums at scale.
- [Context as Code: Build-time Governance in the Era of Infinite Syntax](../../sources/2026-06-13-context-as-code.md) — build-time governance: encode governance rules, architecture patterns and threat models as machine-readable inputs to the generation pipeline — versioned, tested and diff-tracked like code. 'Config drift in constraints is as dangerous as drift in architecture.'
- [Enterprise AI Agent Cost Management Playbook](../../sources/2026-06-13-enterprise-ai-agent-cost-playbook.md) — ROI-first expansion: monitor inference-level ROI before extending access. 'Any investment beyond the playbook is an intentional bet, made only once monitoring is in place.' Unrestricted frontier-model access framed as a cost hazard.
- [Omnigent: Open-sourcing a meta-harness for AI agents](../../sources/2025-06-14-omnigent-meta-harness.md) — Omnigent: contextual policy model (ask permission to git-push if agent downloaded untrusted npm package; pause after 00 spend threshold) — session-state-aware governance rather than static RBAC
