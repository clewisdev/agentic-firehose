# Plan: featured repo widget

A manually-curated "pick of the week" panel on the dashboard. Unlike the trending repos panel (automated, ordered by stars), this is an editorial choice — the owner highlights a specific repo with a short note on why it's worth attention right now.

## Data model

A new file `docs/featured.json` committed manually (not generated by `build.py`):

```json
{
  "repo": "mvanhorn/last30days-skill",
  "note": "Real-time community signal across 7 platforms before client calls. The multi-source synthesis pattern is directly applicable to agentic KB workflows.",
  "set_on": "2026-06-16"
}
```

`build.py` reads this file and includes it in `data.json` as a top-level `featured` key. If the file is absent or the repo isn't found in the repos list, the panel is hidden.

## Dashboard panel design

- Distinct visual treatment — `--brass` accent, labelled "Featured"
- Repo name as a clickable link to GitHub
- Star count from the repos data
- Owner's note (2–3 sentences max)
- "Set on" date
- Positioned: bottom-left (below Synthesis), or as a banner — decide at build time

## Update flow

Owner edits `docs/featured.json` and pushes. GitHub Actions redeploys in ~30s.

## Implementation steps

1. Create `docs/featured.json` with first pick
2. Update `build.py` to read it and include in `data.json`
3. Add `renderFeatured()` function and panel to `index.html`
4. Commit and push
