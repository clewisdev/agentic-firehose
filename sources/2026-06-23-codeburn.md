## Summary

CodeBurn is a working, open-source CLI tool that ingests session files from 31+ AI coding tools (Claude, Cursor, Codex, OpenCode, etc.) and produces real-time cost and token breakdowns—no API keys, no external calls, entirely local analysis.

The repo shows mature engineering: 758 commits, 8.2k stars, active issues/PRs, CI setup, test coverage. Quick-start is `npx codeburn` to launch an interactive dashboard. Installed as a global CLI or macOS menu bar app.

### Core capabilities (from README):

**Overview mode** (`codeburn overview`): Tabular month-at-a-glance summary with cost totals, tokens (in/out/cache-w/cache-r/cache-hit %), calls, sessions, per-tool and per-model breakdown, highest-value days, top projects, per-day table, activity breakdown.

**Optimization scan** (`codeburn optimize`): Pattern detection for waste:
- Files re-read across sessions (same content loaded repeatedly)
- Low read:edit ratios (flagging retries)
- Uncapped BASH_MAX_OUTPUT_LENGTH and trailing noise
- Ghost agents/skills/slash commands in `~/.claude/` never invoked
- Bloated CLAUDE.md files (@-import expansion)
- Cache creation overhead
- Context-heavy sessions with poor token ROI

Each finding includes estimated token/dollar savings and actionable fix (CLAUDE.md line, env var, archival command). Health grade A–F. Findings tracked as new/improving/resolved over 48-hour windows.

**Model comparison** (`codeburn compare`): Interactive picker to evaluate which model is actually better for your work, with cost/performance metrics.

**Dashboard UI**: Arrow-key navigation, time-period switching, date-range filters (`--from`/`--to`), per-provider filtering (`--provider claude`), JSON output option, plaintext mode (`--no-color`).

### Technical detail:

- Reads session files directly from disk (no wrapper/proxy).
- Pricing sourced from LiteLLM, refreshed daily.
- Supports Cursor and OpenCode via automatic better-sqlite3 installation.
- Requires Node.js 22.13+.
- installable via `npm install -g`, `brew`, `bunx`, or `dx`.

## Takeaways

- **Concrete implementation of cost-observation**: Working, deployed tool that solves a real pain point (opacity in multi-tool AI spend) without external dependencies or trust requirements.
- **Session-file analysis pattern**: Demonstrates practical approach to understanding tool behavior by ingesting logs/session data, not instrumentation.
- **Optimization as a core feature, not an afterthought**: `optimize` command is a codified waste-detection engine, not just reporting.
- **Multi-tool support at scale**: 31+ tools suggests serious effort at reverse-engineering and standardizing diverse session formats.
- **Local-first data handling**: Addresses privacy and cost control concerns; pricing refresh suggests dynamic, maintainable pricing source (LiteLLM).
